# Contributing to NEX-GDDP-CMIP6 Downloader

First of all, thank you for taking the time to contribute! 🎉  
Every contribution, big or small, is appreciated.

---

## 🐛 Reporting Bugs

If you found a bug, please open an Issue on GitHub and include:

- Your **operating system** (Windows / macOS / Linux)
- Your **Python version** (`python --version`)
- **Steps to reproduce** the bug
- What you **expected** to happen vs what **actually** happened
- Any **error messages** from the log window

---

## 💡 Suggesting Features

Have an idea? Open an Issue with the label `enhancement` and describe:

- What the feature does
- Why it would be useful
- Any examples or references

---

## 🔧 Submitting Code (Pull Requests)

1. **Fork** the repository on GitHub
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/YOUR-USERNAME/nex-gddp-cmip6-downloader.git
   ```
3. Create a **new branch** for your change:
   ```bash
   git checkout -b fix/my-bug-fix
   ```
4. Make your changes and **test** them
5. **Commit** with a clear message:
   ```bash
   git commit -m "Fix: resolved crash when no ensemble is selected"
   ```
6. **Push** to your fork:
   ```bash
   git push origin fix/my-bug-fix
   ```
7. Open a **Pull Request** on the original repository

---

## ✅ Code Style Guidelines

- Follow **PEP 8** Python style guide
- Use **descriptive variable names**
- Add **comments** for complex logic
- Keep functions **small and focused**
- Test your changes before submitting

---

## 📋 Branch Naming Convention

| Type | Example |
|------|---------|
| Bug fix | `fix/download-retry-crash` |
| New feature | `feature/dark-mode` |
| Documentation | `docs/update-readme` |
| Refactor | `refactor/speed-calculator` |
